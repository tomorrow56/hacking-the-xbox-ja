# 第1章：リバースエンジニアリング入門

> **原著**: *Hacking the Xbox: An Introduction to Reverse Engineering*  
> **著者**: Andrew "bunnie" Huang  
> **出版**: No Starch Press, 2003  
> **本翻訳**: 非公式日本語訳（unofficial Japanese translation）  
> **ライセンス**: CC BY-NC-SA 1.0

---

リバースエンジニアリング (reverse engineering) とは、人工物の構造・機能・動作原理を分析することで、その設計や実装を理解しようとする行為だ。機械から電子機器、ソフトウェアまで、あらゆる人工物がリバースエンジニアリングの対象になりうる。

本章では、リバースエンジニアリングの基本的な概念と手法を紹介し、本書全体を通じて使用するツールと思考の枠組みを準備する。

---

## リバースエンジニアリングとは何か

リバースエンジニアリングを一言で表すなら、「完成品から設計図を復元する試み」だ。

通常の工学（フォワードエンジニアリング）は、設計図や仕様書から出発して、動作する製品を作り上げる。リバースエンジニアリングはその逆方向のプロセスだ。手元にある製品や装置を出発点として、その内部構造や動作メカニズムを解明しようとする。

リバースエンジニアリングが行われる理由はさまざまだ。競合製品の分析、セキュリティ脆弱性の調査、互換性のある製品の開発、文書化されていないシステムの理解、あるいは純粋な知的好奇心など、動機は多岐にわたる。

ハードウェアのリバースエンジニアリングとソフトウェアのリバースエンジニアリングは、使用するツールや手法が異なるが、根底にある論理は同じだ。本書では主にハードウェアを対象とするが、Xboxのファームウェア（ブートROMやカーネル）のリバースエンジニアリングを通じて、ソフトウェアのリバースエンジニアリングにも踏み込む。

---

## ハードウェアのリバースエンジニアリング

### 観察と計測

ハードウェアのリバースエンジニアリングはまず観察から始まる。外観の記録、基板の写真撮影、チップのシルク印刷の読み取り——こうした地道な作業が最初の一歩だ。

次に計測だ。電圧、電流、周波数、タイミングの計測により、回路の動作を定量的に把握する。特に重要なのは、各種バス信号（データバス、アドレスバス、コントロール信号）の観測だ。デジタルオシロスコープやロジックアナライザがここで活躍する。

### コンポーネントの特定

現代の電子機器に搭載されているICチップは、ほとんどの場合、型番やロゴが印字されている。この情報をデータシートと照合することで、各チップの機能・ピン配置・動作仕様を調べることができる。

しかし、意図的に型番を消去したり、カスタムシリコンを使用したりして、解析を困難にしている製品もある。Xboxのある重要なチップもそのようなケースだ。

### バスの解析

コンポーネントを特定したら、それらがどのように接続されているかを把握する必要がある。現代のシステムでは、CPUとメモリ、CPUとI/Oデバイスはバスで接続されている。

PCIバスやISAバス、あるいはメモリバスなど、標準的なバスプロトコルについては公開された仕様が存在する。カスタムバスの場合はより困難だが、タイミング図とデータパターンの分析から、プロトコルを逆に解析できることが多い。

**バス探索の一般的な手順:**

1. バスのクロック信号を特定する
2. データ線とアドレス線を特定する
3. チップセレクト信号・リード/ライト信号を特定する
4. 典型的なトランザクションのタイミングを記録する
5. 記録したデータから既知のプロトコルとのマッチングを試みる

## ソフトウェアのリバースエンジニアリング

ソフトウェアのリバースエンジニアリングでは、コンパイル済みのバイナリコードを逆アセンブルまたはデコンパイルして、元のアルゴリズムやデータ構造を理解しようとする。

### 逆アセンブル (Disassembly)

逆アセンブラは、機械語（バイナリコード）をアセンブリ言語のニーモニックに変換するツールだ。たとえば、x86アーキテクチャのバイナリ列 `55 89 E5` は逆アセンブラによって次のように変換される：

```asm
push   ebp
mov    ebp, esp
```

これは典型的なx86関数プロローグだ。スタックフレームを設定するためのコードで、多くの関数の先頭に現れる。

逆アセンブラだけでは生成されるアセンブリが膨大になることが多く、解析には高度なスキルと経験が必要だ。IDA Pro は業界標準の商用逆アセンブラだが、GNUのobjdumpなどの無償ツールも広く使われる。

### デコンパイル (Decompilation)

デコンパイラはさらに一歩進んで、アセンブリ言語をC言語に近い高水準言語に変換しようとする。完全な復元は原理的に不可能だが（変数名、コメント、コンパイラの最適化情報は失われる）、アルゴリズムの把握には有用だ。

### 動的解析 (Dynamic Analysis)

逆アセンブルやデコンパイルといった静的解析に対して、実際にコードを動かしながら解析するアプローチを動的解析と呼ぶ。デバッガ (debugger) を使って、特定のアドレスにブレークポイントを設定し、レジスタやメモリの状態を確認しながら動作を追跡する。

Xboxのようにセキュリティ機構を持つシステムでは、動的解析の環境を整えること自体がひとつの課題となる。第7章では、Xboxのハードウェアデバッグ環境の構築について詳しく解説する。

## 暗号とリバースエンジニアリング

現代の組み込みシステムはセキュリティ強化のために暗号技術を多用している。Xboxもその例外ではない。

リバースエンジニアリングの過程で暗号化されたデータや鍵導出メカニズムに遭遇することは多い。こうした場面では、暗号理論の知識が不可欠だ。本書では必要に応じて暗号の基礎を解説するが、詳細は専門書に譲る。

特に重要なのは、暗号アルゴリズムそのものを破ろうとするのではなく、その実装や鍵管理の弱点を探るアプローチだ。数学的に安全な暗号も、実装が不適切であれば攻撃が成立する場合がある。Xboxのセキュリティシステムでは、まさにそのような実装上の問題が存在した（第8章で詳述）。

## リバースエンジニアリングのツール

### ハードウェアツール

本書を通じて使用する主なハードウェアツールを紹介する：

**デジタルマルチメータ (DMM)**  
電圧、電流、抵抗の計測に用いる基本的な計測器。電源電圧の確認や断線チェックに使う。

**デジタルオシロスコープ**  
電圧の時間的変化を波形として表示する。クロック信号や非同期シリアル信号の観測に必須だ。

**ロジックアナライザ**  
多チャンネルのデジタル信号をサンプリング・表示する計測器。バスプロトコルの解析に不可欠だ。PCIバスやI2Cバスの全トランザクションを記録し、後から解析できる。

**ハンダごて・ハンダ吸取線**  
基板へのプローブ取り付けや、チップの取り外し・交換に使う。表面実装部品（SMD）を扱うには、先端が細いこて先と適切な拡大鏡が必要だ。

### ソフトウェアツール

**IDA Pro / Ghidra**  
逆アセンブラ・デコンパイラ。IDA Pro は業界標準の商用ツール。GhidraはNSAが開発した無償の代替ツール（本書執筆当時は未リリース）。

**GDB（GNU Debugger）**  
オープンソースのデバッガ。ターゲットシステム上でデバッグサーバを動かし、ホスト側のGDBクライアントから遠隔デバッグする手法（リモートデバッグ）が本書でも登場する。

**hexdump / xxd**  
バイナリファイルを16進数で表示するシンプルなツール。ファームウェアの解析でよく使う。

```
00000020: 0000 0000 0000 0000 0000 0000 0000 0000  ................
00000030: 0000 0000 0000 0000 0000 0000 0000 0000  ................
```

## 法的・倫理的考察

リバースエンジニアリングは強力な技術だが、法的・倫理的な側面について理解しておく必要がある。

### 合法的な目的

リバースエンジニアリングが許容される主な目的：

- **互換性の確保**: 競合製品や標準規格との互換性を確保するためのリバースエンジニアリングは、多くの国で認められている
- **セキュリティ研究**: 脆弱性の発見と報告を目的としたセキュリティリサーチ
- **個人的な研究・教育**: 所有する機器の仕組みを理解するための個人的な研究
- **相互運用性**: 既存システムと連携するソフトウェアの開発

### 注意すべき法的リスク

一方、以下の行為には法的リスクが伴う場合がある：

- 著作権で保護されたコードのコピーと再配布
- デジタルコンテンツ保護 (DRM) の回避（DCMAなど）
- 研究成果の公開に際した企業秘密の漏洩

本書の執筆者 bunnie 自身が、このような法的問題に直面した経験を持つ。プロローグで述べた MIT と Microsoft との一件がその典型だ。

リバースエンジニアリングを実践する際は、目的の正当性と、公開する情報の影響を常に意識する必要がある。本書では第11章「ハッキングの倫理と法律」でこの問題を詳しく論じる。

## Xboxのリバースエンジニアリング計画

本書の残りの章では、Xboxを題材に以上の手法を実践する。最終的な目標は、Xboxで任意のコードを実行できる環境（つまり、Microsoft が署名したゲームソフト以外のプログラムも動かせる状態）を作ることだ。

この目標を達成するためには：

1. Xboxのハードウェアアーキテクチャを理解する（第3章）
2. ブートプロセスと暗号化スキームを解析する（第4章、第8章）
3. ブートROMの内容を取得する（第4章）
4. セキュリティチェックを回避してカスタムコードを実行する（第5章、第6章）
5. ファームウェアを改造してより高い制御を得る（第9章、第10章）

各ステップで使うツールと手法は、他の組み込みシステムにも応用できる汎用的なものだ。Xboxはたまたま手頃で普及したプラットフォームだったというだけで、本書で学ぶスキルはあなたが直面するどんなシステムにも活かせるはずだ。
